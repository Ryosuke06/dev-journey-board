# Implementation Plan

> このタスク計画はTDD前提で進める。
> **Red** は「先に失敗するテストを書く」、**Green** は「最小実装でテストを通す」、**Refactor** は「テストが通ったまま整理する」を意味する。
> 各サブタスクは、実装前に「背景」「受け入れ条件」「やらないこと」「備考」を確認できる粒度で記載する。
> `_Requirements:` は `requirements.md` の受け入れ条件IDに対応する。

- [ ] 1. TDDで進めるための基盤を整える

- [x] 1.1 React/Vite/TypeScriptとテスト実行基盤を用意する

  - **受け入れ条件:** React SPAがログインなしで起動し、開発サーバー、ビルド、型チェック、単体テスト、E2Eテストのnpm scriptsが実行可能である。
